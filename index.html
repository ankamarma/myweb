High-Availability Enterprise Topology

Comprehensive Infrastructure, Protocol Design, and Redundancy Validation

1. Infrastructure & System Components

To achieve a resilient design capable of supporting enterprise workloads, the entire logical infrastructure is emulated on dedicated physical hardware using advanced bare-metal and simulation tools:

  • Hardware Platform: Cisco UCS C220 M4 High-Density Rack Server, offering the heavy compute capabilities needed to run multiple network operating system nodes concurrently.
  • Virtualization Host: VMware ESXi Type-1 Hypervisor, providing dedicated resource scheduling, memory allocation, and virtual switching isolation.
  • Simulation Orchestrator: GNS3 (Graphical Network Simulator-3), hosting fully virtualized router and security kernels directly mapping to authentic device software.
  • End-User Client Nodes: Includes UbuntuDesktopGuest20.04.4-2 and Windows7-1 virtual workstations configured within downstream access-layer loops to validate network paths.

2. Network Topology Architecture

The network architecture deploys a highly available, multi-tier layout structured to completely eliminate single points of failure from the edge down to the access endpoints:

  • Dual-Homed WAN Edge: Features connections to two discrete Internet Service Providers (ISP1 and ISP2) using independent transit subnets. Boundary control is distributed across Edge Routers R1 and R2.
  • Demilitarized / Transit Layer (VLAN 10): An isolated external-facing network loop linking edge routers to the firewall interfaces. To safeguard connectivity, intermediate switches FW-SW1 and FW-SW2 carry cross-connected paths linked over a Link Aggregation Control Protocol (LACP) trunk.
  • Stateful Security Tier: Implements a pair of Cisco Adaptive Security Appliances (ASA-FW1 and ASA-FW2). ASA-FW1 acts as the Primary Active firewall, whereas ASA-FW2 remains in a Hot Standby state. They continuously mirror connection tables across a dedicated physical FAILOVER-LINK (192.168.100.0/30).
  • Resilient Aggregation Core: Powered by Cisco Nexus switches (NEX-SW13 and NEX-SW2) running Multi-Chassis EtherChannel technology to present a unified loop-free backbone routing directly into the internal production zones (VLAN 20).
  • Segmented Access Tiers: Downstream switches (LAN-SW1 and LAN-SW2) deliver secure access directly to physical endpoints, systematically parsing departmental broadcasts.

3. Implemented Protocol Framework

To manage optimal traffic flow and sub-second convergence, a sophisticated layer of switching and routing protocols is woven tightly throughout the environment:

Core Protocol Implementation

  • BGP (Border Gateway Protocol): Handles exterior routing path vector decisions. Edge nodes run external BGP peerings—R1 peers with ISP1 (ASN-62001) under ASN-63001, and R2 peers with ISP2 (ASN-62002)—guaranteeing deterministic multi-homed provider visibility.
  • OSPF (Open Shortest Path First): Operates as the internal gateway routing backbone protocol across all routers, firewall interfaces, and core Nexus distribution nodes to automatically map internal topological transit paths.
  • VRRP (Virtual Router Redundancy Protocol): Deployed over the edge interface space to bundle R1 and R2. It advertises a shared Virtual Gateway IP of 192.168.10.1 to the external transit switches.
  • HSRP (Hot Standby Router Protocol): Provisioned exclusively across the Cisco Nexus core switch tier to distribute highly available first-hop default gateways across internal production zones.
  • vPC (Virtual Port Channel): Configured between the Nexus switches to group multiple discrete interfaces across distinct chassis into logical bundles, bypassing rigid Spanning Tree blocking states while enabling all links to forward active traffic.
  • LACP (Link Aggregation Control Protocol): Standardized throughout all core inter-switch trunk connections to aggregate bandwidth, maximize link density, and provide dynamic link failure detection.

VLAN Fabric Structural Mapping

  • VLAN 10 (OUTSIDE Context): Bound to network segment 192.168.20.0/24 for transit handling.
  • VLAN 20 (INSIDE Context): Bound to network segment 192.168.10.0/24 for secure corporate traffic.
