Enterprise Multi-Tier Redundant Network Architecture

This repository showcases a highly resilient, enterprise-grade multi-tier network architecture built to eliminate single points of failure. The design integrates dual-homed ISP edges, stateful firewall clustering, and multi-chassis link aggregation to deliver reliable outbound transit and bulletproof internal segmentation.

Infrastructure, Hardware & Modeling Tools

The entire environment was simulated and rigorously validated using professional network emulation software hosted on bare-metal virtualization infrastructure.

  • Compute Hardware: Cisco Unified Computing System (UCS) C220 M4 Rack Server
  • Hypervisor: VMware ESXi (vSphere)
  • Network Emulation Platform: GNS3 (Graphical Network Simulator-3)
  • End-User Workstations: Ubuntu Desktop Client (20.04.4 LTS) and Windows 7 Enterprise

Configured Protocols & Network Matrix

Traffic handling is dynamically governed by a blend of external and internal routing methods combined with first-hop redundancy protocols:

  • Border Gateway Protocol (BGP): Established across the ISP boundaries between autonomous systems ASN-62001, ASN-62002, and ASN-63001.
  • Open Shortest Path First (OSPF): Deployed as the interior gateway routing protocol (IGP) interconnecting Edge Routers ⇄ ASA Firewalls ⇄ Core Nexus Switches.
  • Virtual Router Redundancy Protocol (VRRP): Configured on the edge paths (R1/R2) sharing a virtual IP layer of 192.168.10.1.
  • Hot Standby Router Protocol (HSRP): Deployed at the Core Nexus layer to orchestrate default gateway resilience for internal VLAN groups.
  • Link Aggregation Control Protocol (LACP) & vPC: Applied systematically across firewall interconnects, distribution nodes, and access rings to scale uplinks and eliminate spanning-tree loops.

Logical Traffic Subnets & Segmentations

  • Vlan 10: Outside / Demilitarized Zone (DMZ Subnet: 192.168.20.0/24)
  • Vlan 20: Inside / Secure Corporate Backbone (LAN Subnet: 192.168.10.0/24)

High Availability (HA) Failover Testing

To verify absolute continuous uptime, the infrastructure was dynamically tested by injecting sudden physical and logical fault assertions. Continuous link state tests verified that LAN users (Ubuntu / Windows clients) never dropped access to public namespaces (8.8.8.8):

  • Scenario A: Complete Edge Router / Primary ISP Drop PASS
    Result: Primary path fell flat; BGP reconverged globally and VRRP safely shifted traffic to the backup node.
  • Scenario B: Firewall Distribution Switch Failure PASS
    Result: Interconnecting LACP lanes adapted on-the-fly; traffic automatically shifted around the dark switch over the alternative uplink channels.
  • Scenario C: Active Security Appliance Failure (ASA-FW1) PASS
    Result: Active firewall crashed; stateful session database replicated across the FAILOVER-LINK (192.168.100.0/30) let ASA-FW2 grab ongoing sockets instantly.
  • Scenario D: Core Layer Switch Outage (NEX-SW1) PASS
    Result: Core switch went offline; vPC pairs and HSRP tracking moved internal workloads seamlessly to NEX-SW2.
